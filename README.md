# Proover

## Общий механизм работы программы
В этом разделе README описывается общий механизм работы программы под названием “autoProover”. Она строит дерево секвенций (структуру, где каждая вершина — это текущее состояние доказательства), а затем пытается доказать каждую “ветвь” дерева при помощи набора логических правил. Ниже перечислены основные идеи:

Начальный секвент (начальная пара “левая часть ⊢ правая часть”) добавляется в дерево.

На каждом шаге программа выбирает секвент и пытается вывести его правую часть из левой части, используя:
• Правило Modus Ponens
• Теорему о дедукции (позволяет «раскрыть» импликации в правой части)
• Правило избавления от отрицаний (перекидывает ¬A из левой части в правую или наоборот)
Если удаётся свести любую ветвь дерева к аксиоматическому случаю (когда левая и правая части пересекаются), секвент считается доказанным.

## Веб представление программы
Для данной программы также был реализован удобный фронтенд, облегчающий взаимодействие с программой. Для запуска локально понадобится Flask:
